systemd is the system and service manager for Linux. It runs as PID 1, brings up the rest of the system, and manages the full lifecycle of services — from boot to shutdown. systemd provides a unified toolset for service management, structured logging, network configuration, device management, and secure credential handling.

## Get started

<Steps>
  <Step title="Check your systemd version">
    Verify systemd is running and check the version installed on your system.

    ```bash
    systemctl --version
    ```
  </Step>
  <Step title="List running services">
    See all active units and their current state.

    ```bash
    systemctl list-units --type=service --state=running
    ```
  </Step>
  <Step title="Enable a service at boot">
    Enable a service to start automatically on next boot, then start it now.

    ```bash
    systemctl enable --now myservice.service
    ```
  </Step>
  <Step title="Inspect logs">
    Query the systemd journal for logs from a specific service.

    ```bash
    journalctl -u myservice.service -f
    ```
  </Step>
</Steps>

<Note>
  Most systemctl commands that modify system state require root privileges or membership in appropriate groups. Use `sudo` where needed.
</Note>

Modern Linux daemons do not need to daemonize themselves, manage PID files, or redirect their own stdio. systemd handles all of that. A new-style daemon stays in the foreground, writes to stdout/stderr, and communicates its lifecycle to systemd through a small set of well-defined interfaces.

## Service unit basics

Every managed process needs a `.service` unit file that tells systemd how to start, stop, and supervise it.

```ini /etc/systemd/system/myapp.service
[Unit]
Description=My Application
After=network.target

[Service]
Type=notify
ExecStart=/usr/bin/myapp --config /etc/myapp/config.toml
ExecReload=/bin/kill -HUP $MAINPID
Restart=on-failure
User=myapp
StateDirectory=myapp
RuntimeDirectory=myapp
LogsDirectory=myapp

[Install]
WantedBy=multi-user.target
```

### Service type

The `Type=` directive tells systemd how to detect that startup is complete.

| Type | When systemd considers the service ready |
|---|---|
| `simple` | Immediately after `fork()` — no feedback from the process |
| `exec` | After `execve()` succeeds — catches missing binaries |
| `notify` | After the process sends `READY=1` via `sd_notify()` |
| `notify-reload` | Like `notify`, plus uses `RELOADING=1` to track reloads |
| `forking` | After the parent process exits — legacy, avoid for new code |

`Type=notify` or `Type=notify-reload` is the recommended choice for any daemon that takes non-trivial time to initialize.

### Exec directives

```ini
[Service]
ExecStart=/usr/bin/myapp
ExecReload=/bin/kill -HUP $MAINPID
ExecStop=/usr/bin/myapp --shutdown
```

- `ExecStart=` — the command to run. Must be an absolute path.
- `ExecReload=` — command to trigger a configuration reload. With `Type=notify-reload`, this can be omitted entirely if the service sends `RELOADING=1` when it receives `SIGHUP`.
- `ExecStop=` — optional cleanup command. systemd sends `SIGTERM` first and waits `TimeoutStopSec=` before escalating to `SIGKILL`.

### Restart policies

```ini
[Service]
Restart=on-failure        # restart only on non-zero exit or signal
RestartSec=5s
```

| Value | Restarts when |
|---|---|
| `no` | Never (default) |
| `on-success` | Clean exit only |
| `on-failure` | Non-zero exit, signal, timeout, or watchdog failure |
| `on-abnormal` | Signal, timeout, or watchdog failure (not non-zero exit) |
| `always` | Always, regardless of exit status |

## Readiness notification

The recommended way to signal that a service has finished initializing is via the `sd_notify()` protocol. systemd opens a Unix datagram socket and passes its path in the `$NOTIFY_SOCKET` environment variable. The service sends newline-delimited key=value messages to that socket.

<Note>
You must set `Type=notify` (or `Type=notify-reload`) in the service unit, otherwise systemd ignores all notification messages.
</Note>

### Notification messages

| Message | Meaning |
|---|---|
| `READY=1` | Startup complete; service is ready to handle requests |
| `RELOADING=1` | Beginning a configuration reload |
| `STOPPING=1` | Beginning clean shutdown |
| `STATUS=<text>` | Human-readable status string shown in `systemctl status` |
| `WATCHDOG=1` | Watchdog keepalive ping |
| `MAINPID=<pid>` | Update the tracked main PID |
| `ERRNO=<n>` | Report an errno value on failure |

### Using libsystemd

Link against `libsystemd` and include `<systemd/sd-daemon.h>`:

```c
#include <systemd/sd-daemon.h>

// Signal readiness
sd_notify(0, "READY=1");

// Signal readiness with status text
sd_notifyf(0, "READY=1\nSTATUS=Listening on port 8080\nMAINPID=%lu",
           (unsigned long) getpid());

// Signal start of reload
sd_notify(0, "RELOADING=1");

// Signal readiness again after reload completes
sd_notify(0, "READY=1");

// Signal clean shutdown
sd_notify(0, "STOPPING=1");
```

### Self-contained implementation

If you cannot depend on `libsystemd`, the protocol is simple enough to implement directly. The following examples are standalone implementations that carry no external dependencies.

<CodeGroup>

```c notify.c
/* SPDX-License-Identifier: MIT-0 */

/* Implement the systemd notify protocol without external dependencies.
 * Supports both readiness notification on startup and on reloading,
 * according to the protocol defined at:
 * https://www.freedesktop.org/software/systemd/man/latest/sd_notify.html
 * This protocol is guaranteed to be stable as per:
 * https://systemd.io/PORTABILITY_AND_STABILITY/ */

#define _GNU_SOURCE 1
#include <errno.h>
#include <inttypes.h>
#include <signal.h>
#include <stdbool.h>
#include <stddef.h>
#include <stdlib.h>
#include <stdio.h>
#include <string.h>
#include <sys/socket.h>
#include <sys/un.h>
#include <time.h>
#include <unistd.h>

#define _cleanup_(f) __attribute__((cleanup(f)))

static void closep(int *fd) {
  if (!fd || *fd < 0)
    return;

  close(*fd);
  *fd = -1;
}

static int notify(const char *message) {
  union sockaddr_union {
    struct sockaddr sa;
    struct sockaddr_un sun;
  } socket_addr = {
    .sun.sun_family = AF_UNIX,
  };
  size_t path_length, message_length;
  _cleanup_(closep) int fd = -1;
  const char *socket_path;

  /* Verify the argument first */
  if (!message)
    return -EINVAL;

  message_length = strlen(message);
  if (message_length == 0)
    return -EINVAL;

  /* If the variable is not set, the protocol is a noop */
  socket_path = getenv("NOTIFY_SOCKET");
  if (!socket_path)
    return 0; /* Not set? Nothing to do */

  /* Only AF_UNIX is supported, with path or abstract sockets */
  if (socket_path[0] != '/' && socket_path[0] != '@')
    return -EAFNOSUPPORT;

  path_length = strlen(socket_path);
  /* Ensure there is room for NUL byte */
  if (path_length >= sizeof(socket_addr.sun.sun_path))
    return -E2BIG;

  memcpy(socket_addr.sun.sun_path, socket_path, path_length);

  /* Support for abstract socket */
  if (socket_addr.sun.sun_path[0] == '@')
    socket_addr.sun.sun_path[0] = 0;

  fd = socket(AF_UNIX, SOCK_DGRAM|SOCK_CLOEXEC, 0);
  if (fd < 0)
    return -errno;

  if (connect(fd, &socket_addr.sa, offsetof(struct sockaddr_un, sun_path) + path_length) != 0)
    return -errno;

  ssize_t written = write(fd, message, message_length);
  if (written != (ssize_t) message_length)
    return written < 0 ? -errno : -EPROTO;

  return 1; /* Notified! */
}

static int notify_ready(void) {
  return notify("READY=1");
}

static int notify_reloading(void) {
  /* A buffer with length sufficient to format the maximum UINT64 value. */
  char reload_message[sizeof("RELOADING=1\nMONOTONIC_USEC=18446744073709551615")];
  struct timespec ts;
  uint64_t now;

  /* Notify systemd that we are reloading, including a CLOCK_MONOTONIC timestamp
   * in usec so that the program is compatible with a Type=notify-reload service. */

  if (clock_gettime(CLOCK_MONOTONIC, &ts) < 0)
    return -errno;

  now = (uint64_t) ts.tv_sec * 1000000ULL + (uint64_t) ts.tv_nsec / 1000ULL;

  if (snprintf(reload_message, sizeof(reload_message),
               "RELOADING=1\nMONOTONIC_USEC=%" PRIu64, now) < 0)
    return -EINVAL;

  return notify(reload_message);
}

static int notify_stopping(void) {
  return notify("STOPPING=1");
}

static volatile sig_atomic_t reloading = 0;
static volatile sig_atomic_t terminating = 0;

static void signal_handler(int sig) {
  if (sig == SIGHUP)
    reloading = 1;
  else if (sig == SIGINT || sig == SIGTERM)
    terminating = 1;
}

int main(int argc, char **argv) {
  struct sigaction sa = {
    .sa_handler = signal_handler,
    .sa_flags = SA_RESTART,
  };
  int r;

  /* Setup signal handlers */
  sigemptyset(&sa.sa_mask);
  sigaction(SIGHUP, &sa, NULL);
  sigaction(SIGINT, &sa, NULL);
  sigaction(SIGTERM, &sa, NULL);

  /* Do more service initialization work here … */

  /* Now that all the preparations steps are done, signal readiness */
  r = notify_ready();
  if (r < 0) {
    fprintf(stderr, "Failed to notify readiness to $NOTIFY_SOCKET: %s\n", strerror(-r));
    return EXIT_FAILURE;
  }

  while (!terminating) {
    if (reloading) {
      reloading = false;

      r = notify_reloading();
      if (r < 0) {
        fprintf(stderr, "Failed to notify reloading to $NOTIFY_SOCKET: %s\n", strerror(-r));
        return EXIT_FAILURE;
      }

      /* Do some reconfiguration work here … */

      r = notify_ready();
      if (r < 0) {
        fprintf(stderr, "Failed to notify readiness to $NOTIFY_SOCKET: %s\n", strerror(-r));
        return EXIT_FAILURE;
      }
    }

    /* Do some daemon work here … */
    sleep(5);
  }

  r = notify_stopping();
  if (r < 0) {
    fprintf(stderr, "Failed to report termination to $NOTIFY_SOCKET: %s\n", strerror(-r));
    return EXIT_FAILURE;
  }

  /* Do some shutdown work here … */
  return EXIT_SUCCESS;
}
```

```python notify.py
#!/usr/bin/python
# SPDX-License-Identifier: MIT-0
#
# Implement the systemd notify protocol without external dependencies.
# Supports both readiness notification on startup and on reloading,
# according to the protocol defined at:
# https://www.freedesktop.org/software/systemd/man/latest/sd_notify.html
# This protocol is guaranteed to be stable as per:
# https://systemd.io/PORTABILITY_AND_STABILITY/

import errno
import os
import signal
import socket
import sys
import time

reloading = False
terminating = False

def notify(message):
    if not message:
        raise ValueError("notify() requires a message")

    socket_path = os.environ.get("NOTIFY_SOCKET")
    if not socket_path:
        return

    if socket_path[0] not in ("/", "@"):
        raise OSError(errno.EAFNOSUPPORT, "Unsupported socket type")

    # Handle abstract socket.
    if socket_path[0] == "@":
        socket_path = "\0" + socket_path[1:]

    with socket.socket(socket.AF_UNIX, socket.SOCK_DGRAM | socket.SOCK_CLOEXEC) as sock:
        sock.connect(socket_path)
        sock.sendall(message)

def notify_ready():
    notify(b"READY=1")

def notify_reloading():
    microsecs = time.clock_gettime_ns(time.CLOCK_MONOTONIC) // 1000
    notify(f"RELOADING=1\nMONOTONIC_USEC={microsecs}".encode())

def notify_stopping():
    notify(b"STOPPING=1")

def reload(signum, frame):
    global reloading
    reloading = True

def terminate(signum, frame):
    global terminating
    terminating = True

def main():
    global reloading, terminating

    # Set up signal handlers.
    signal.signal(signal.SIGHUP, reload)
    signal.signal(signal.SIGINT, terminate)
    signal.signal(signal.SIGTERM, terminate)

    # Do any other setup work here.

    # Once all setup is done, signal readiness.
    notify_ready()

    while not terminating:
        if reloading:
            reloading = False

            # Support notifying the manager when reloading configuration.
            # This allows accurate state tracking as well as automatically
            # enabling 'systemctl reload' without needing an ExecReload= line.
            notify_reloading()

            # Do some reconfiguration work here.

            notify_ready()

        # Do the real work here ...
        time.sleep(5)

    notify_stopping()

if __name__ == "__main__":
    sys.stdout.reconfigure(line_buffering=True)
    main()
```

## Watchdog

For long-running services, systemd can kill and restart a process that stops responding. Enable this with `WatchdogSec=` in the unit file. The service must then send `WATCHDOG=1` at least once every `WatchdogSec=` interval. Sending once every half the interval is recommended.

```ini
[Service]
Type=notify
WatchdogSec=30s
```

In the service code, check whether watchdog is active using `sd_watchdog_enabled()`, then send periodic pings:

```c
#include <systemd/sd-daemon.h>
#include <stdint.h>

uint64_t watchdog_usec = 0;

// Check if watchdog is expected (call once at startup)
if (sd_watchdog_enabled(0, &watchdog_usec) > 0) {
    // Send WATCHDOG=1 every watchdog_usec/2 microseconds
}

// In your main loop, periodically:
sd_notify(0, "WATCHDOG=1");
```

`$WATCHDOG_USEC` and `$WATCHDOG_PID` are set by systemd in the service environment. `sd_watchdog_enabled()` reads these variables and returns the timeout in microseconds.

A service can also trigger the watchdog action on demand if it detects an internal error:

```c
sd_notify(0, "WATCHDOG=trigger");
```

This triggers the same action as a missed keepalive, without requiring `WatchdogSec=` to be set.

## Signal handling

<Steps>
  <Step title="Handle SIGTERM for clean shutdown">
    systemd sends `SIGTERM` when stopping a service. Send `STOPPING=1` before beginning any shutdown work, then exit cleanly.

    ```c
    static void on_sigterm(int sig) {
        sd_notify(0, "STOPPING=1");
        // begin shutdown...
    }
    ```
  </Step>
  <Step title="Handle SIGHUP for reload">
    systemd sends `SIGHUP` when `systemctl reload` is invoked (if `ExecReload=` is set to send `SIGHUP`). Send `RELOADING=1` with a monotonic timestamp, re-read configuration, then send `READY=1` again.

    ```c
    static void on_sighup(int sig) {
        sd_notify(0, "RELOADING=1\nMONOTONIC_USEC=...");
        // reload config...
        sd_notify(0, "READY=1");
    }
    ```
  </Step>
  <Step title="Exit with a meaningful code">
    systemd records the exit code and uses it to decide whether to restart the service. Exit 0 for success, non-zero for failure. The `Restart=on-failure` policy restarts only on non-zero exits and signals.
  </Step>
</Steps>

## Logging

New-style services do not need to open syslog or manage log files. Write to stdout and stderr; systemd's journal captures both automatically.

```c
fprintf(stdout, "Server started on port 8080\n");
fprintf(stderr, "Warning: config file not found, using defaults\n");
```

Structured log fields can be added using the journal protocol directly via `sd_journal_print()` or `sd_journal_send()`, but plain text to stderr is sufficient for most services.

<Tip>
Set `StandardOutput=journal` and `StandardError=journal` in the `[Service]` section to make the journal capture explicit. These are the defaults when running under systemd, so you usually do not need to set them.
</Tip>

## Environment variables and directories

systemd sets several environment variables that services can use to find their managed directories and credentials.

| Variable | Set when | Path |
|---|---|---|
| `CREDENTIALS_DIRECTORY` | `LoadCredential=` or `SetCredential=` is used | Read-only directory of credential blobs |
| `RUNTIME_DIRECTORY` | `RuntimeDirectory=` is set | `/run/<name>` — cleared on service stop |
| `STATE_DIRECTORY` | `StateDirectory=` is set | `/var/lib/<name>` — persisted across restarts |
| `CACHE_DIRECTORY` | `CacheDirectory=` is set | `/var/cache/<name>` |
| `LOGS_DIRECTORY` | `LogsDirectory=` is set | `/var/log/<name>` |
| `CONFIGURATION_DIRECTORY` | `ConfigurationDirectory=` is set | `/etc/<name>` |

Enable these in the unit file:

```ini
[Service]
StateDirectory=myapp
RuntimeDirectory=myapp
CacheDirectory=myapp
```

Access them in the service:

```c
const char *state_dir = getenv("STATE_DIRECTORY");  // e.g. /var/lib/myapp
const char *runtime_dir = getenv("RUNTIME_DIRECTORY");  // e.g. /run/myapp
const char *creds = getenv("CREDENTIALS_DIRECTORY");  // e.g. /run/credentials/myapp.service
```

## User services vs system services

systemd supports per-user service managers in addition to the system-wide one. A unit file placed under `~/.config/systemd/user/` is managed by the user's instance of systemd.

<Tabs>
  <Tab title="System service">
    ```bash
    # Install
    sudo cp myapp.service /etc/systemd/system/
    sudo systemctl daemon-reload
    sudo systemctl enable --now myapp

    # Manage
    sudo systemctl status myapp
    sudo systemctl restart myapp
    journalctl -u myapp
    ```
  </Tab>
  <Tab title="User service">
    ```bash
    # Install
    mkdir -p ~/.config/systemd/user/
    cp myapp.service ~/.config/systemd/user/
    systemctl --user daemon-reload
    systemctl --user enable --now myapp

    # Manage
    systemctl --user status myapp
    journalctl --user -u myapp
    ```
  </Tab>
</Tabs>

<Note>
User services run only while the user is logged in, unless `loginctl enable-linger <username>` is set, which keeps the user manager alive after logout.
</Note>

## Security hardening

Modern service units support extensive sandboxing without any code changes to the daemon. Common options:

```ini
[Service]
# Run as a dedicated unprivileged user
User=myapp
Group=myapp

# Restrict filesystem access
ProtectSystem=strict
ProtectHome=true
ReadWritePaths=/var/lib/myapp

# Network and privilege restrictions
PrivateNetwork=false
NoNewPrivileges=true
PrivateTmp=true

# Restrict system calls
SystemCallFilter=@system-service
CapabilityBoundingSet=
```

These restrictions are enforced by the kernel and do not require the service to do anything special.
